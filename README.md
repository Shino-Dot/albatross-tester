## 💡 概要 (For Everyone & HR)
**「複雑なマニュアルを、誰でも迷わず完遂できる最高の武器へ」**

アルバトロスは、カスタマーサポートや現場のオペレーターが、故障診断において「次に何を案内すべきか」を瞬時に判断するために開発された、故障診断チャート・プラットフォームです。

単なるフローチャートのデジタル化に留まらず、利用者の習熟度（初級・中級・上級）に合わせてUIを動的に最適化。**「技術的制約を言い訳にせず、現場のホスピタリティを最大化する」**という設計思想のもと、エンジニアリングと実務利便性の両立を追求しました。

### 💎 コアバリュー
- **レベル別アダプティブUI:** 同一の診断ロジックを、ユーザーのスキルに合わせて「丁寧な案内」から「効率的なタイル表示」まで自動切替。
- **データドリブンな改善:** 全ての診断プロセスをログ化。どのステップで改善したか、あるいは未解決だったかを可視化し、マニュアル自体の品質向上に寄与。
- **パーソナライズされた操作感:** 文字サイズ変更やテーマカラー切り替えなど、長時間の業務でも疲れないアクセシビリティ機能を搭載。

## 🚀 AI活用戦略 (Strategic Engineering)
本プロジェクトでは、AIを単なるコード生成ツールではなく、**「アーキテクチャの妥当性検証」**と**「複雑なデータ構造の整理」**におけるパートナーとして活用しました。

- **多階層モデルの設計レビュー:** チャート種別、ステップ、セッションログという複雑なリレーションシップにおいて、クエリパフォーマンスを損なわないモデル構成をAIと共に検証。
- **非同期保存ロジックの最適化:** JavaScript(Fetch API)とDjango間のデータ送受信において、CSRF対策とユーザーを待たせない非同期処理の最適解を高速に導き出しました。

## ✨ 技術的な挑戦とこだわり (For Tech & HR)

### 🌟 ① 現場の「魂」を実装した「上級モード」UI
**課題:** 習熟したオペレーターにとって、順を追うだけのチャートは「非効率」であるという現場のジレンマがありました。
**解決:** 全行程をタイル状に俯瞰し、自由な順番で診断を進められる「上級モード」を独力で設計・実装。
- **リスクの可視化:** 信号機コンセプト（赤・黄・青）に基づき、試すべき操作のリスクを直感的に判別可能。
- **自由選択フロー:** 任意の順序で操作を記録でき、タイムスタンプを自動刻印。効率化と証跡管理を高次元で両立させました。

### 🌟 ② デプロイ沼の克服：執念のインフラエンジニアリング
**課題:** Fly.ioへのデプロイ時、PostgreSQLのSSL接続拒否（SSL SYSCALL error）などの複数の環境依存エラーに直面しました。
**解決:** ログの徹底的な解析により、Fly.ioが自動生成する `DATABASE_URL` のパラメータ競合が原因であることを特定。
- **動的パッチの実装:** `settings.py` 内でURL文字列を動的に置換し、SSL接続を強制確立させるロジックを自ら構築。
- 「動かない」を「動かす」まで決して諦めない、エンジニアとしてのデバッグ能力と粘り強さを証明しました。

### 🌟 ③ 徹底したユーザー・ファーストなUX設計
**課題:** 長時間の診断業務におけるオペレーターの疲弊を軽減する必要がありました。
**解決:** `localStorage` を活用した設定の永続化と、ストレスフリーなUIを実現。
- **アクセシビリティ:** 4段階の文字サイズ変更と、パステル調を中心とした5種類のカラーテーマ切り替え機能を実装。
- **案内性の向上:** 回答後、次の質問へ視覚的に誘導するスムーズな自動スクロール処理や、詳細解説アコーディオンを搭載。

## 🔧 主要機能
- **動的レベル切替:** 初級（丁寧な説明）・中級（要点のみ）・上級（全体俯瞰）のリアルタイム切替。
- **診断ログ履歴:** 過去の全診断プロセスを詳細に記録。期間検索やソート機能を完備。
- **セキュアなアカウント管理:** 認証を挟んだ二段階退会フロー、論理削除によるデータ整合性の保護。
- **管理画面カスタマイズ:** Django Adminを魔改造し、チャート構造を視覚的に管理可能に。
