# Git超入門！プロジェクト管理のための虎の巻

## A. 新しいプロジェクトで、最初に一回だけやること

これは、新しいPCで初めてGitを使う時や、新しいプロジェクトをGit管理下に置く時に、**一番最初に一回だけ**やればOKな設定だよ！

### 1. Gitに自分の名前とメアドを教える（PCで最初の1回だけ！）

ターミナルで、下のコマンドを実行する。`""`の中は、自分の情報に書き換えること。

```shell
# ユーザー名の設定
git config --global user.name "Your Name"

# メールアドレスの設定
git config --global user.email "your_email@example.com"
```

### 2. プロジェクトをGitの管理下におく

プロジェクトのルートフォルダ（`manage.py`がある場所）で、ターミナルを開いて、下のコマンドを**順番に**実行する。

#### ① `.gitignore` ファイルの作成

Gitに無視してほしいファイル（`venv`, `db.sqlite3`など）をリストアップする、超重要なファイル。
プロジェクトのルートに、`.gitignore`という名前でファイルを作成し、中身を記述する。（Python/Django用のテンプレートをコピペするのがオススメ！）

#### ② リポジトリの初期化

「このフォルダを、Gitの管理場所にするぞ！」という宣言。

```shell
git init
```

#### ③ 全ファイルをステージング

「今あるファイルを、全部、最初の保存の準備リストに入れて！」という命令。

```shell
git add .
```

#### ④ 最初のコミット（保存）

「最初の状態」として、メッセージ付きで保存する。

```shell
git commit -m "Initial commit: プロジェクトの初期設定"
```
これで、最初のタイムカプセルが完成！

---

## B. 日々の開発で、繰り返しやること

一度最初の設定が終わったら、あとはこの作業の繰り返し！

### 1. コードを書く・修正する

いつも通り、新しい機能を追加したり、バグを修正したりする。

### 2. 変更内容を確認する

「前回の保存から、どのファイルがどう変わったかな？」って確認する。

```shell
git status
```
赤色で表示されてるのが、変更があったファイルだよ。

### 3. 変更したファイルを、保存の準備リストに入れる

「この変更を、次のタイムカプセルに入れる準備をして！」という命令。

```shell
# 特定のファイルだけ準備する場合
git add accounts/views.py

# 変更があったファイルを、全部まとめて準備する場合（よく使う！）
git add .
```

### 4. 変更内容を、メッセージ付きで保存（コミット）する

「準備ができたファイルを、こういう変更内容で保存します！」という命令。
`-m "..."`のメッセージは、**「何をしたか」が分かりやすいように書く**のが、超絶大事！

```shell
# 良い例
git commit -m "feat: ログ一覧に、日付フィルター機能を追加"
git commit -m "fix: アカウント削除後に、ログイン画面に飛ばされるバグを修正"
git commit -m "refactor: LogListViewのコードを、もっと読みやすく整理"

# 悪い例（何したか分からん！）
git commit -m "修正"
git commit -m "いろいろやった"
```

**この「①コード書く → ②確認 → ③準備 → ④保存」のサイクルを、キリがいい単位で（例えば、1つの機能が完成したら、とか）繰り返していくのが、基本的なGitの使い方だよ！**
